# POS UI Redesign — Implementation Plan

## Context

Frontend: `frontend/`, React 18 + TypeScript + Vite + Tailwind CSS 3.
Build: `tsc --noEmit && vite build` — type errors block the build.
`tsconfig.json` has `noUnusedLocals` + `noUnusedParameters: true` — every import must be consumed.
No icon library — all icons are inline SVG with `stroke="currentColor"`, `strokeWidth={2}`.
No existing `@layer components` — this plan introduces the first shared utility classes.

## Constraints

- Inline SVG only (no icon library).
- German convention for currency: amount then `€` symbol (e.g. `7,47 €`). Replace all `EUR` text in values with `€`. Column headers drop the currency annotation entirely (just `Preis`, `Gesamt`).
- Low-stock color shifts from `orange-600`/`orange-500` to `amber-500`/`amber-100`/`amber-700` throughout.
- Don't touch `ProductFormModal.tsx` — it works and is out of scope.
- Don't touch `Scanner.tsx` logic — only visual if needed (not required by this plan).

## Validation

After all edits:
1. Run `cd frontend && npx tsc --noEmit` — must pass with zero errors. Check for unused imports (especially if any SVG-related imports are accidentally added — there shouldn't be any since SVGs are inline JSX).
2. Run `cd frontend && npm run build` — must succeed (tsc + vite build).
3. Visual spot-check: `npm run dev`, navigate all three tabs (Kasse, Inventar, Bestellungen), verify:
   - Cart images are larger (20x20)
   - Cart sticks on scroll (desktop/lg viewport)
   - Tab icons render
   - Nav is sticky
   - Search has a magnifying glass icon
   - Notification close is an SVG X
   - Low-stock badge is amber pill, not orange `!`
   - All currency shows `€` suffix, no `EUR` text remains
   - Product cards have separated barcode line

## Risks

- **Unused imports:** `tsc --noEmit` will fail if any import is not consumed. Since all SVGs are inline JSX (no imports needed), this should be safe — but double-check if any task accidentally introduces an unused import.
- **Sticky cart overflow:** If the cart list grows very long, `lg:max-h-[calc(100vh-8rem)] lg:overflow-y-auto` on the `<ul>` prevents the sticky container from exceeding viewport height. The total + checkout button remain outside the scroll area (they're siblings of `<ul>`, not children).
- **`@layer components` + `@apply`:** Tailwind 3.4 supports this. The `input` class uses `@apply` with `focus:` variants — ensure PostCSS/Tailwind processes correctly (it will, since `@tailwind` directives already work).
